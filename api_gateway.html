DANH SÁCH CHI TIẾT ENDPOINTS CHO GATEWAYS

Mô tả cấu trúc toàn bộ Request, Response phục vụ lập trình Backend Spring Boot và nhúng thiết bị phần cứng phần Gateway.

1. NHÓM API QUẢN TRỊ (ADMIN / DASHBOARD INTERFACE)
POST /api/v1/gateways Admin Only

Chức năng: Khởi tạo bản ghi Gateway phần cứng mới, liên kết chặt chẽ khóa ngoại với bảng Station.

Thông số Đầu Vào (Request Body - JSON)
Tên trườngKiểu dữ liệuBắt buộcRàng buộc dữ liệu / Ý nghĩa
stationIdString (UUID)CóPhải tồn tại trong bảng stations, độ dài cố định 36 ký tự.
codeStringCóMã Gateway duy nhất (Unique), không trống, tối đa 100 ký tự.
serialNumberStringKhôngSố Serial phần cứng máy, duy nhất (Unique) nếu truyền.
modelStringKhôngDòng sản phẩm chip/mạch thiết bị (Tối đa 100 ký tự).
firmwareVersionStringKhôngPhiên bản phần mềm gốc cài đặt trong máy.
ipAddressStringKhôngĐịa chỉ IP mạng LAN hoặc WAN (Regex kiểm tra chuỗi định dạng IPv4).
macAddressStringKhôngĐịa chỉ MAC vật lý của card mạng (Regex bọc chuẩn: 00:1A:3F...).
Mẫu Request Body
{
  "stationId": "b182cb34-01fa-47cb-bc12-9988ccffaadd",
  "code": "GW-LANGELEMENT-01",
  "serialNumber": "SN-2026-X89",
  "model": "Advantech-ICAM",
  "firmwareVersion": "v1.0.0",
  "ipAddress": "192.168.10.15",
  "macAddress": "A1:B2:C3:D4:E5:F6"
}
Mẫu Response (200 OK)
{
  "status": "SUCCESS",
  "code": 200,
  "message": "Thao tác thành công",
  "data": {
    "id": "7fa3c150-bba2-4b2a-8991-66df3bfa4cc6",
    "code": "GW-LANGELEMENT-01",
    "status": "OFFLINE",
    "isDeleted": false
  }
}
PUT /api/v1/gateways Admin Only

Chức năng: Sửa đổi thông tin cấu hình phần cứng. API tích hợp cơ chế Merge chống ghi đè trường null của MapStruct.

Thông số Đầu Vào (Request Body - JSON)
Tên trườngKiểu dữ liệuBắt buộcRàng buộc dữ liệu / Ý nghĩa
idString (UUID)CóKhóa chính ID của Gateway cần sửa để hệ thống quét DB tra cứu.
stationIdString (UUID)KhôngTruyền lên khi muốn di chuyển Gateway sang Trạm khác quản lý.
modelStringKhôngTên dòng máy mới thay đổi. Nếu để trống hệ thống giữ nguyên dòng cũ.
firmwareVersionStringKhôngCập nhật mốc phiên bản hệ điều hành.
ipAddressStringKhôngĐịa chỉ IP mạng mới setup lại.
macAddressStringKhôngĐịa chỉ MAC mới thay thế card mạng.
GET /api/v1/gateways User/Admin

Chức năng: Tìm kiếm nâng cao, phân trang động danh sách thiết bị. Tích hợp trực tiếp GenericSpecification qua Reflection.

Tham số trên URL (Query Parameters)
Tên tham sốKiểu dữ liệuMặc địnhÝ nghĩa lọc
pageInteger0Số thứ tự trang muốn lấy dữ liệu (Bắt đầu từ số 0).
sizeInteger10Số lượng bản ghi hiển thị trên một trang.
codeStringnullTìm kiếm gần đúng (Like %code%) theo mã hiệu thiết bị.
stationIdStringnullLọc toàn bộ danh sách thiết bị trực thuộc một trạm cụ thể.
statusString (Enum)nullLọc trạng thái vận hành phần cứng: ONLINE, OFFLINE, ERROR.
DELETE /api/v1/gateways/{id} Admin Only

Chức năng: Xóa mềm thiết bị phần cứng ra khỏi hệ thống để bảo toàn log dữ liệu đo đạc lịch sử cũ.

URL mẫu: /api/v1/gateways/7fa3c150-bba2-4b2a-8991-66df3bfa4cc6

2. NHÓM API KẾT NỐI TRẠM VẬT LÝ (IOT TELEMETRY CHIP-TO-SERVER)
POST /api/v1/gateways/heartbeat Device Only

Chức năng: Nhận tín hiệu Ping định kỳ duy trì trạng thái sóng mạng từ mạch vi xử lý vật lý đặt tại các tỉnh gửi về.

Dữ liệu mạch cứng gửi lên (JSON)
Tên trườngKiểu dữ liệuBắt buộcÝ nghĩa xử lý của hệ thống
codeStringCóMã Gateway dán nhãn cố định trong chip để Server tìm ra bản ghi.
ipAddressStringKhôngĐịa chỉ IP hiện thời của trạm (Cập nhật nếu trạm dùng mạng IP động).
firmwareVersionStringKhôngBáo cáo phiên bản phần mềm hiện tại để hệ thống lưu dấu vết.
Mạch Cứng đẩy lên (Mỗi 30 giây)
{
  "code": "GW-HANOI-01",
  "ipAddress": "172.16.5.99",
  "firmwareVersion": "v1.0.2"
}
Server ghi nhận và phản hồi
{
  "status": "SUCCESS",
  "code": 200,
  "message": "Heartbeat acknowledged",
  "data": {
    "status": "ONLINE",
    "lastSeen": "2026-06-02T09:34:19Z"
  }
}
POST /api/v1/gateways/report-error Device Only

Chức năng: Cơ chế ngắt khẩn cấp. Khi phần cứng gặp sự cố chập mạch, sụt áp hoặc mất cảm biến, nó lập tức đẩy lệnh này về để Server đổi trạng thái sang hệ báo động ERROR.

Tên trườngKiểu dữ liệuBắt buộcÝ nghĩa
codeStringCóMã thiết bị báo lỗi.
errorCodeStringCóMã lỗi chuẩn hóa. Ví dụ: ERR_OVERHEAT, ERR_VOLTAGE_LOW.
errorMessageStringKhôngNội dung lỗi chi tiết trích từ log hệ thống của máy vật lý.
POST /api/v1/gateways/firmware/check Device Only

Chức năng: Thiết bị vật lý gửi lên để kiểm tra xem trên hệ thống máy chủ trung tâm có bản cập nhật hệ điều hành (Firmware) mới hay không.

Mẫu Request thiết bị gửi lên
{
  "code": "GW-HANOI-01",
  "currentFirmware": "v1.0.2"
}
Mẫu Response Server trả về đường dẫn OTA
{
  "status": "SUCCESS",
  "code": 200,
  "data": {
    "hasUpdate": true,
    "newVersion": "v1.0.3",
    "downloadUrl": "https://iot.server.com/bin/gw-v1.0.3.bin",
    "checksum": "sha256-e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855"
  }
}
3. NHÓM API BÁO CÁO & THỐNG KÊ (DASHBOARD ANALYTICS)
GET /api/v1/gateways/analytics/summary User/Admin

Chức năng: Tổng hợp dữ liệu tức thời từ cơ sở dữ liệu để kết xuất ra tổng số lượng máy đang chạy ngon lành, tắt nguồn hay đang báo động phục vụ vẽ đồ thị tròn trên màn hình giám sát trung tâm.

Dữ liệu Response nhận về (JSON)
{
  "status": "SUCCESS",
  "code": 200,
  "message": "Thao tác thành công",
  "data": {
    "totalDevices": 150,
    "onlineCount": 132,
    "offlineCount": 15,
    "errorCount": 3
  }
}
